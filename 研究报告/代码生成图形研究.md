## 一、核心转换原理

### 1. AST 解析流程

代码转图形块的核心流程分为两个方向：

#### 正向：Python → 图形块（代码可视化）
Python 源码首先被解析为 **AST（抽象语法树）**，然后 AST 的每个节点被映射为对应的图形块。

**关键项目：PythonBlockly**（CoderBotOrg）
- GitHub: https://github.com/CoderBotOrg/PythonBlockly
- 描述：Translating Python code back to Blockly blocks reconstructing the AST using Skulpt
- 技术栈：Skulpt（浏览器端 Python 解析器）+ Blockly

**PythonBlockly 核心解析流程：**
```javascript
// 1. 使用 Skulpt 解析 Python 源码为 AST
parse = Sk.parse(filename, python_source);
ast = Sk.astFromParse(parse.cst, filename, parse.flags);

// 2. 遍历 AST 节点，转换为 Blockly XML
var converted = this.convert(ast);

// 3. convert() 方法根据 node._astname 分发到具体处理器
PythonToBlocks.prototype.convert = function(node, is_top_level) {
    return this[node._astname](node, is_top_level);
}
```

| Python AST 节点 | 图形块类型 (Blockly) |
|---|---|

| `Assign` (a = 0) | `variables_set` |

| `While` | `controls_while` |
| `For` (range) | `coderbot_repeat` |
| `For` (foreach) | `controls_forEach` |
| `FunctionDef` | `procedures_defnoreturn` |
| `Return` | `procedures_return` |
| `Print` | `text_print` / `text_print_multiple` |

| `Compare` | `logic_compare` / `logic_isIn` |
| `BoolOp` | `logic_operation` (AND/OR) |
| `UnaryOp` (Not) | `logic_negate` |
| `Call` | `procedures_callreturn` |

| `Str` | `text` / `string_multiline` |

### 3. 嵌套结构的图形表达

#### 顺序结构
Blockly 使用 `<next>` 元素链接相邻块：
```javascript
function nestChild(child) {
    if (root == null) {
        root = child;
        current = child;
    } else {
        var nextElement = document.createElement("next");
        nextElement.appendChild(child);
        current.appendChild(nextElement);
        current = child;
    }
}
```

#### 条件结构（If/Else）
使用 mutation 扩展参数表示多分支：
```javascript
return block("controls_if_better", node.lineno, {},
    {"IF0": this.convert(test)},  // 条件输入
    {"inline": "false"},
    {"@elseif": elseifCount, "@else": elseCount},  // mutation 扩展
    {"DO0": this.convertBody(body), "ELSE": ...}   // 语句容器
);
```

#### 循环结构
- `For` 循环：使用 `coderbot_repeat`（固定次数）或 `controls_forEach`（遍历列表）
- `While` 循环：使用 `controls_while`
- 语句体通过 `DO` 参数传入 `convertBody()`

#### 函数定义
```javascript
return block("procedures_defnoreturn", node.lineno, {
    "NAME": this.identifier(name)
}, {}, {"inline": "false"},
    {"arg": this.arguments_(args)},  // 参数 mutation
    {"STACK": this.convertBody(body)}  // 函数体
);
```

## 二、代码生成器架构

### 1. 前端：图形块 → JSON AST

Blockly 工作区中的图形块被序列化为 **XML** 格式（Blockly Workspace XML），包含：
- `type`：块类型名称
- `line_number`：源码行号（用于反向映射）
- `fields`：文本字段值
- `values`：嵌套的输入值（可以是另一个块）
- `statements`：语句容器（包含子块序列）
- `mutation`：可变参数（用于支持 else/elif 分支数量等）

```xml
<block type="controls_if_better" line_number="5">
  <mutation elseif="1" else="1"></mutation>
  <value name="IF0">
    <block type="logic_compare">
      <field name="OP">GT</field>
      <value name="A"><block type="variables_get"><field name="VAR">x</field></block></value>
      <value name="B"><block type="math_number"><field name="NUM">10</field></block></value>
    </block>
  </value>
  <statement name="DO0">...</statement>
  <statement name="ELSE">...</statement>
</block>
```

### 2. 中端：JSON AST → Python AST

图形块 XML 被解析并转换为 Python AST（`ast` 模块）：

```javascript
// Blockly → Python AST（以 For 循环为例）
if (iter.func.id.v === "range") {
    // range() 调用 → For 循环 AST
    return block("coderbot_repeat", node.lineno, {"TIMES": iter.args[0].n.v}, ...);
} else {
    // 普通迭代 → ForEach AST
    return block("controls_forEach", node.lineno, {}, {"LIST": this.convert(iter), ...});
}
```

### 3. 后端：Python AST → 可执行代码

Blockly 自带代码生成器框架，每个块类型注册一个生成函数：

```javascript
python.pythonGenerator.forBlock['block_type'] = function(block, generator) {
    // 1. 获取子块的值代码
    var value_input = generator.valueToCode(block, 'input_name', python.Order.ATOMIC);
    
    // 2. 获取子语句
    var statements_body = generator.statementToCode(block, 'DO');
    
    // 3. 组装代码
    var code = '...' + value_input + '...\n';
    
    // 4. 返回代码（表达式需返回 [code, order] 数组）
    return [code, python.Order.ATOMIC];
};
```

**优先级（Order）系统：**
```javascript
// 用于处理运算符优先级，确保生成代码加括号正确
python.Order.ATOMIC        // 原子级（数字、变量）
python.Order.CLASSBINDING   // 类绑定
python.Order.COLLECTION     // 容器
python.Order.STRING_CONVERSION  // 字符串转换
// ... 更多级别
```

## 三、效率优化原则

### 1. 代码块合并优化

**原理：** 相邻同类型操作合并为一个块，减少块数量。

**示例：**
- 连续多个 `print()` 调用 → 合并为 `text_print_multiple`
- 连续赋值 `a = 0; b = 0;` → 在 Blockly 中垂直排列的多个 `variables_set` 块

**实现：** 在 `convertBody()` 中检测相邻块的类型：
```javascript
if (is_top_level && skipped_line && visitedFirstLine) {
    addPeer(newChild);  // 间隔行，作为同级 peer
} else if (is_top_level && !previousWasStatement) {
    addPeer(newChild);
} else {
    nestChild(newChild);  // 嵌套到当前块内
}
```

### 2. 冗余消除

**死代码删除（Dead Code Elimination）：**
- 未使用的变量赋值
- 不可达代码（如 `return` 后的语句）
- 无用注释块

**实现策略：** 在 AST 到块的转换过程中，通过 `line_number` 属性追踪源码位置，跳过无实际作用的节点。

### 3. 死代码删除

**PythonBlockly 中的实现：**
```javascript
PythonToBlocks.prototype.Delete = function(targets) {
    // Delete 语句在 Blockly 中无对应块（隐式操作）
    return null;
}
```

Import 语句同样返回 `null`：
```javascript
PythonToBlocks.prototype.Import = function(node) {
    return null;  // import 在 Blockly 中隐式处理
}
```

### 4. 循环展开优化

**适用场景：** 固定次数循环 `for i in range(N)` 在 N 较小时展开为顺序块。

**注意：** 过度展开会导致块数量爆炸，需设置阈值（如 N ≤ 4）。

### 3. 多线程/并行任务代码块

**实现思路：** 使用 `procedures_callreturn` 块表示并行任务：

```javascript
// 未知函数调用 fallback
var argumentsNormal = {};
var argumentsMutation = {"@name": name};
for (var i = 0; i < args.length; i++) {
    argumentsNormal["ARG"+i] = this.convert(args[i]);
    argumentsMutation["*"+i] = this.convert(args[i]);
}
return block("procedures_callreturn", node.lineno, {}, argumentsNormal, {
    "inline": "true"
}, argumentsMutation);
```

**并行执行表示：** 可通过 `spawn` 或 `async` 函数调用实现，映射为特定的 CoderBot 块（如事件生成器）。

```javascript
PythonToBlocks.prototype.convertBody = function(node, is_top_level) {
    // 遍历每个语句节点
    for (var i = 0; i < node.length; i++) {
        var newChild = this.convertStatement(node[i], originalSourceCode, is_top_level);
        
        // 根据上下文决定嵌套或同级
        if (is_top_level && skipped_line && visitedFirstLine) {
            addPeer(newChild);   // 作为同级 peer
        } else {
            nestChild(newChild); // 嵌套到当前块
        }
    }
}
```
