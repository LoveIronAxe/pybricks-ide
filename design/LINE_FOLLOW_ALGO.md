# 单光巡线算法详解

> 本文档用于 IDE 中生成和调参，涵盖算法原理、P控制器参数、扫描频率、速度限制、阈值标定以及 IDE 参数输入界面设计。

---

## 1. 单光巡线原理

### 1.1 反射率原理

ColorSensor 的 `reflection()` 返回值代表**地面反射光强度**，范围 **0~100**：

| 反射率值 | 含义 |
|---------|------|
| **0** | 完全黑（不反光） |
| **100** | 完全白（全反射） |

> 实际传感器在黑线上典型值约为 **15~30**，在白纸上典型值约为 **50~80**，具体取决于光照条件。

### 1.2 黑白判断逻辑

通过**阈值（threshold）**将连续反射率值二值化为"在线上"或"在线外"：

```
if reflection > threshold → 认为是白色（偏左/偏右）
if reflection < threshold → 认为是黑色（在线上）
```

**阈值的计算方式：**

```
threshold = (white_ref + black_ref) / 2
```

- `white_ref`：在白色区域实测的反射率
- `black_ref`：在黑色线段上实测的反射率

> ⚠️ **不同场地光线阈值差异很大**，建议每次比赛/训练前重新标定。

### 1.3 差速转向物理原理

Pybricks 差速转向通过设置左右电机的**转速差**实现转向：

| turn_rate 值 | 转向方向 | 物理效果 |
|-------------|---------|---------|
| **> 0** | 右转 | 左轮加速 + 右轮减速 |
| **< 0** | 左转 | 左轮减速 + 右轮加速 |
| **= 0** | 直行 | 两轮同速 |

**驱动代码示意：**

```python
left_motor.speed(target_speed_left + turn_rate / 2)
right_motor.speed(target_speed_right - turn_rate / 2)
```

## 2. P控制器参数详解

### 2.1 核心公式

```
error = reflection - threshold
turn_rate = Kp × error
```

| 变量 | 含义 | 方向判断 |
|-----|------|---------|
| `error > 0` | 偏白 → 偏左 | turn_rate > 0 → 右转修正 |
| `error < 0` | 偏黑 → 偏右 | turn_rate < 0 → 左转修正 |
| `error = 0` | 正好在中心 | 直行 |

### 2.2 Kp 比例系数

**推荐范围：0.3 ~ 4.0**

| Kp 值 | 效果描述 | 调参建议 |
|-------|---------|---------|
| **太小的 Kp（< 0.5）** | 机器人反应迟钝，修正力度弱，**容易冲出线** | 适当调大 |
| **合适的 Kp（0.8~2.0）** | 响应灵敏，转向修正及时，过弯流畅 | 推荐起始值 |
| **太大的 Kp（> 3.0）** | 机器人剧烈抖动、**左右振荡**，无法稳定循线 | 适当调小 |

### 2.3 调参口诀

> **"小了就加，大了就减，调到小车稳在黑线上小幅左右摆动为止"**

**推荐调参流程：**
1. 从 `Kp = 1.0` 开始
2. 观察机器人循线行为
3. 如果冲出线 → 加 Kp（+0.2）
4. 如果抖动振荡 → 减 Kp（-0.2）
5. 重复直到稳定

## 3. 扫描频率（wait_ms）

### 3.1 推荐范围：5 ~ 20ms

**采样频率计算：**

```
每秒采样次数 = 1000 / wait_ms
```

| wait_ms | 每秒采样次数 | 评价 |
|---------|-------------|------|
| **5 ms** | 200 Hz | 极快，传感器压力较大 |
| **10 ms** | 100 Hz | 推荐默认值 |
| **20 ms** | 50 Hz | 较慢，但稳定 |
| **30 ms** | 33 Hz | 偏慢，容易冲出线 |
| **50 ms** | 20 Hz | ⚠️ 非常慢，不推荐 |

### 3.2 wait_ms 过大/过小的影响

| 情况 | 问题 |
|-----|------|
| **wait_ms 太小（< 5ms）** | 可能内存溢出、传感器采样不稳定、CPU 负载过高 |
| **wait_ms 太大（> 30ms）** | 机器人反应太慢，循线滞后，**容易冲出线** |

## 4. 速度限制

### 4.1 电机速度参数

| 参数 | 值 | 说明 |
|-----|-----|------|
| `MAX_MOTOR_SPEED` | **1180 deg/s** | 中速电机实测最大转速 |
| `MAX_SPEED` | **1062 deg/s** | 安全上限（90% 最大值） |
| `MIN_SPEED` | **-354 deg/s** | 反转速度上限（-30%） |

> 实际循线时，`base_speed` 会自动限幅在 `MIN_SPEED` ~ `MAX_SPEED` 之间，防止电机超速或反转过快损坏。

### 4.2 速度与稳定性的关系

- **速度越快**，允许的修正时间窗口越小，对 Kp 和 scan_freq 要求越高
- **低速（< 300 deg/s）**：容错性高，Kp 偏小也能稳定
- **高速（> 800 deg/s）**：需要更大的 Kp 和更高的扫描频率

## 5. 阈值（Threshold）标定

### 5.1 标定公式

```
threshold = (white_ref + black_ref) / 2
```

### 5.2 典型值范围

| 场地条件 | 典型 threshold 值 |
|---------|------------------|
| 光照稳定 | 30 ~ 50 |
| 光照偏暗 | 20 ~ 35 |
| 光照强烈 | 45 ~ 60 |

> ⚠️ **不同场地光线阈值差异很大**，强烈建议在比赛/训练前使用 IDE 提供的标定功能实测白纸和黑线的反射率，再计算阈值。

### 6.2 各参数输入控件规格

| 参数 | 控件类型 | 最小值 | 最大值 | Step | 默认值 | 特殊提示 |
|-----|---------|-------|-------|-----|-------|---------|
| **Kp** | 数字输入框 | 0.3 | 4.0 | 0.1 | 1.0 | 无 |
| **扫描频率** | 数字输入框 | 5 | 50 | 1 | 10 | 见下方警告规则 |
| **速度** | 数字输入框 | 0 | 1062 | 10 | 500 | 无 |
| **阈值** | 数字输入框 | 0 | 100 | 1 | 40 | 无 |

### 6.3 扫描频率警告规则

| 条件 | 警告级别 | 显示颜色 | 提示文案 |
|-----|---------|---------|---------|
| `wait_ms > 30` | ⚠️ 橙色提示 | 橙色背景 | "偏慢，建议 ≤30ms，机器人可能冲出线" |
| `wait_ms < 5` | 🔴 红色警告 | 红色背景 | "过快，建议 ≥5ms，可能导致采样不稳定" |
| `5 ≤ wait_ms ≤ 30` | ✅ 绿色正常 | 绿色背景 | "频率正常" |

### 6.4 参数联动预览

IDE 应实时展示当前参数代入公式后的效果示例，帮助用户理解参数含义：

```
┌─────────────────────────────────────────┐
│ 📊 公式预览                               │
│                                          │
│ error = reflection - threshold           │
│    = 35 - 40 = -5 (偏黑，向右偏离)        │
│                                          │
│ turn_rate = Kp × error                  │
│    = 1.0 × (-5) = -5                    │
│    → turn_rate < 0，机器人左转修正        │
│                                          │
│ 采样频率: 1000 / 10ms = 100 Hz          │
└─────────────────────────────────────────┘
```

### 6.5 标定功能

提供一键标定功能，自动读取白纸和黑线的反射率并计算阈值：

```
┌─────────────────────────────────────────┐
│ 📐 阈值标定                              │
│                                          │
│ 1. 将传感器放在白纸上                     │
│    [ 点击测量白色 ] → white_ref = 68     │
│                                          │
│ 2. 将传感器放在黑线上                     │
│    [ 点击测量黑色 ] → black_ref = 18     │
│                                          │
│ 3. 自动计算阈值                          │
│    threshold = (68 + 18) / 2 = 43       │
│                                          │
│    [ 应用阈值 ]                          │
└─────────────────────────────────────────┘
```
