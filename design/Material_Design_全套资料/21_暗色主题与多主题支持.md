# Material Design 3 — 暗色主题与多主题支持

## 1. 亮色主题 vs 暗色主题色彩对比

Material Design 3（Material You）定义了完整的色彩系统，在亮色和暗色主题下呈现出不同的视觉特征。

| 角色 | 亮色主题 | 暗色主题 | 说明 |
|------|----------|----------|------|
| **Primary** | 主品牌色（高饱和） | 保持色调，降低明度 | Primary 不"反转"，而是调暗 |
| **On Primary** | 白色/极浅色 | 深色（保证对比度） | 确保文字在主色上有足够对比 |
| **Surface** | 白色 `#FFFFFF` | 深灰/近黑色系 | Surface 1/2/3/5 提供层次 |
| **On Surface** | 深色文字 | 浅色文字 | 文字颜色随主题反转 |
| **Surface Variant** | 浅灰 | 深灰 | 次级表面区域 |
| **Outline / Outline Variant** | 中灰 | 略浅灰 | 边框、分割线颜色 |
| **Error** | 红色系 | 保持红色系 | 错误色通常保持不变或略调暗 |
| **Inverse Surface** | — | 浅色浮层 | 用于对话框等需要"反转"的场景 |
| **Inverse On Surface** | — | 深色文字 | 配合 Inverse Surface 使用 |
| **Inverse Primary** | — | 浅色主色 | 用于强调的"反转主色" |
| **Primary Container** | 浅主色背景 | 深主色背景 | 容器类组件 |
| **On Primary Container** | 深主色文字 | 浅主色文字 | 容器内文字 |
| **Secondary / Tertiary** | 与 Primary 类似处理 | 降低明度，保持色调 | 次要色同样调暗处理 |

### 色彩系统核心原则

```
亮色主题  →  Surface 越浅越好，文字越深越好
暗色主题  →  Surface 越深越好，文字越浅越好
主色处理  →  保持色调(Hue)，只降低明度(Lightness)
```

## 2. 暗色主题的色彩反转规则（Surface 1/2/3/5 区分层次）

Material Design 3 在暗色主题中通过 Surface 1/2/3/5 四个层级建立纵深层次，替代亮色主题中 Surface 1-5（越浅越亮）的逻辑。

### 亮色主题 Surface 层级（从上到下由浅到深）

| 层级 | 亮度 | 典型用途 |
|------|------|----------|
| Surface | `#FFFFFF` | 最顶层，浮动元素 |
| Surface 1 | `#F5F5F5` | 主背景（最低层） |
| Surface 2 | `#EEEEEE` | 次级背景 |
| Surface 3 | `#E8E8E8` | 卡片、面板 |
| Surface 4 / 5 | `#E0E0E0` | 分割区域（极少用） |

### 暗色主题 Surface 层级（从上到下由浅到深）

| 层级 | 亮度 | 典型用途 |
|------|------|----------|
| Surface | `#121212`（或 `#1C1B1F` for M3） | 最底层背景 |
| **Surface 1** | `#1E1E1E` 或 `#1A1A1A` | 主背景（最低层） |
| **Surface 2** | `#232323` | 次级背景/抽屉背景 |
| **Surface 3** | `#282828` | 卡片、浮层、模态框背景 |
| **Surface 5** | `#2E2E2E` | 最高层浮层（如 FAB 的上一层） |

> **关键规则**：暗色主题中 Surface 层级的逻辑是 **Surface 1 最暗，Surface 5 最浅**，与亮色主题相反！这是因为暗色主题需要越"底层"越暗，浮层越浅。

### Elevation 与 Surface 的映射关系

```
elevation 0  →  Surface (最低层，最暗)
elevation 1  →  Surface + 极薄白色叠加层
elevation 2  →  Surface 1
elevation 3  →  Surface 2
elevation 4  →  Surface 3
elevation 5+ →  Surface 5
```

暗色主题通过在 Surface 上叠加极薄的白色/浅色半透明层来模拟海拔高度，模拟光影效果：
- 亮色主题用阴影表示高度（越亮越高）
- 暗色主题用浅色叠加层表示高度（越浅越高）

## 3. 暗色主题的主色如何调整（保持色调，降低明度）

### 色彩调整算法

暗色主题的主色调整遵循 **HSL 调整原则**：

```
亮色主题 Primary:  HSL(H, S%, L%)
暗色主题 Primary:  HSL(H, S%, L - ΔL)  ← 色调不变，明度降低

ΔL（明度差值）通常为 20-35%
```

### M3 官方推荐的 Primary 调整方式

| 组件 | 亮色主题 | 暗色主题 |
|------|----------|----------|
| Primary | 100% 饱和度 | 饱和度略微降低（约 90%） |
| Primary Container | 浅主色（如 `Primary/10%` 透明度） | 深主色（如 `Primary/30%` 透明度） |
| On Primary Container | 深主色文字 | 浅主色文字 |

### 具体数值示例（以紫色 `Purple` 品牌色为例）

```
--md-sys-color-primary:          #6750A4  (亮色)
--md-sys-color-on-primary:       #FFFFFF

--md-sys-color-primary-dark:     #D0BCFF  (暗色 Primary，保持色调)
--md-sys-color-on-primary-dark:  #381E72  (暗色 On Primary，确保对比)
```

### 辅助色（Secondary / Tertiary）同样处理

```
Secondary: HSL(H, S%, L%)  →  HSL(H, S * 0.8, L - ΔL)
Tertiary:  HSL(H, S%, L%)  →  HSL(H, S * 0.9, L - ΔL)
```

## 4. 本项目 IDE 暗色主题配色建议

基于陛下的 PyBricks IDE 场景，我们采用 **M3 暗色主题规范**，推荐以下配色：

### 品牌主色建议

考虑到 PyBricks 是乐高机器人编程 IDE，建议使用 **蓝色系** 作为主色：

| 用途 | 色值 | 说明 |
|------|------|------|
| Primary | `#5B9BD5` | 乐高蓝色系，亮色主题主色 |
| On Primary | `#FFFFFF` | 白色文字 |
| Primary Container | `#D6E8FA` | 浅主色容器 |
| On Primary Container | `#0A2740` | 容器内深色文字 |

| 用途 | 色值 | 说明 |
|------|------|------|
| Primary (暗色) | `#A8CFF5` | 调暗后保持蓝色调 |
| On Primary (暗色) | `#0D3A5C` | 深蓝文字确保对比 |
| Primary Container (暗色) | `#1E3A5F` | 深主色容器背景 |
| On Primary Container (暗色) | `#D6E8FA` | 浅色文字 |

### Surface 层级建议

```css
/* 暗色主题 Surface 层级 */
--color-surface-dark-1: #121212;   /* 最底层背景 */
--color-surface-dark-2: #1E1E1E;   /* 次级背景 */
--color-surface-dark-3: #262626;   /* 卡片/面板 */
--color-surface-dark-4: #2E2E2E;   /* 高层浮层 */
--color-surface-dark-5: #383838;   /* 最高层（如 FAB 背景） */
```

### 文字颜色建议

```css
/* 暗色主题文字 */
--color-on-surface-dark:      #E6E6E6;   /* 主文字 */
--color-on-surface-variant-dark: #B0B0B0; /* 次级文字 */
--color-outline-dark:          #5C5C5C;   /* 边框 */
--color-outline-variant-dark:  #3C3C3C;   /* 次级边框 */
```

### 代码编辑器背景建议

IDE 代码编辑器区域通常比通用 Surface 更深：

```css
/* 代码编辑器专用 */
--color-editor-background-dark: #1A1A2E;  /* 代码块深蓝黑 */
--color-editor-gutter-dark:     #252540;  /* 行号栏 */
--color-editor-line-highlight-dark: #2A2A4A; /* 当前行高亮 */
```

## 7.陛的项目 IDE 的主题方案建议

### 推荐方案：双主题 + 用户偏好持久化

陛下的 PyBricks IDE 采用 **Electron + Web 前端**架构，推荐以下方案：

```
┌─────────────────────────────────────────────┐
│  主题切换架构                                │
├─────────────────────────────────────────────┤
│  1. 读取用户偏好（localStorage / userConfig）│
│  2. 若无偏好 → 跟随系统 prefers-color-scheme │
│  3. 用户可手动切换：亮色 / 暗色 / 跟随系统   │
│  4. 切换方式：手动类名 .dark-theme 覆盖      │
│     （比 media query 更灵活，支持强制指定）   │
└─────────────────────────────────────────────┘
```

### 主题默认策略

| 场景 | 默认行为 |
|------|----------|
| 首次使用（无偏好） | 跟随操作系统 `prefers-color-scheme` |
| 用户主动切换 | 记住用户选择，持久化到配置 |
| 显式选择"亮色" | 强制亮色，忽略系统偏好 |
| 显式选择"暗色" | 强制暗色，忽略系统偏好 |
| 显式选择"跟随系统" | 动态跟随系统偏好变化 |

### 实现建议

#### 1. HTML 结构

```html
<!-- 主题类名放在 <html> 或 <body> 上 -->
<html class="theme-system">
  <!-- 或 theme-light / theme-dark -->
</html>
```

#### 2. JavaScript 主题管理器

```javascript
// theme-manager.js
const ThemeManager = {
  STORAGE_KEY: 'pybricks-theme-preference',
  
  // 三种模式
  MODES: ['system', 'light', 'dark'],
  
  // 初始化
  init() {
    const saved = localStorage.getItem(this.STORAGE_KEY) || 'system';
    this.apply(saved);
    this.listenSystemPreference();
  },
  
  // 切换主题
  setTheme(mode) {
    if (!this.MODES.includes(mode)) return;
    localStorage.setItem(this.STORAGE_KEY, mode);
    this.apply(mode);
  },
  
  // 应用主题到 DOM
  apply(mode) {
    const html = document.documentElement;
    html.classList.remove('theme-system', 'theme-light', 'theme-dark');
    html.classList.add(`theme-${mode}`);
  },
  
  // 监听系统偏好变化
  listenSystemPreference() {
    window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', (e) => {
      const saved = localStorage.getItem(this.STORAGE_KEY);
      if (saved === 'system') {
        // 动态刷新页面或重新应用主题
        this.apply('system');
        // 手动触发 CSS 变量更新
        document.dispatchEvent(new CustomEvent('theme-system-change', { 
          detail: { isDark: e.matches } 
        }));
      }
    });
  }
};
```

### 重要注意事项

1. **避免 FOUT（无样式闪烁）**：在 `<head>` 中尽早注入主题检测脚本，防止页面加载时闪屏
2. **CSS 变量覆盖顺序**：CSS 变量优先级低于媒体查询内的同名变量（当两者选择器相同时，源码顺序决定），注意代码结构
3. **对比度合规**：暗色主题的 On Surface 文字颜色不低于 WCAG AA 标准（4.5:1），Primary 上文字不低于 3:1
4. **编辑器特殊处理**：IDE 代码编辑器背景建议比 Surface 更深，但不要过深（避免纯黑 `#000000`，影响代码可读性），推荐蓝黑色系
5. **测试覆盖**：在两种主题下分别测试代码高亮、图表、状态指示器等所有视觉元素
