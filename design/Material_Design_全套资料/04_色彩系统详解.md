# 色彩系统详解

> Material Design 3 通过色调、色相、饱和度、明度的系统化数学关系，从单一主色自动生成完整调色板。

---

## 一、核心概念：色调、色相、饱和度、明度

### 1.1 四个维度的关系

| 概念 | 定义 | 在 MD3 中的作用 |
|------|------|----------------|
| **色调 (Hue)** | 颜色的基本属性，决定是红、绿、蓝等 | MD3 使用 13 种固定色调：Red、Orange、Yellow、Lime、Green、Cyan、Blue、Purple、Magenta、Pink、Teal、Indigo、Brown |
| **色相 (Chroma)** | 颜色的纯度/鲜艳程度 | 同一色调内的深浅变化 |
| **明度 (Lightness/Tonal Elevation)** | 颜色的明亮或暗沉程度 | MD3 定义 13 级明度（0-100），数字越大越亮 |
| **饱和度 (Saturation)** | 颜色的浓淡 | 与色相共同决定颜色的鲜艳程度 |

### 1.2 MD3 色调轮与明度等级

MD3 调色板每个色调有 13 个色阶，编号从 0（最暗）到 100（最亮）：

```
色阶 0   ██  最暗（接近黑色）
色阶 10  ██
色阶 20  ██
色阶 30  ██
色阶 40  ██  ← 主角色阶（Primary 常用）
色阶 50  ██
色阶 60  ██
色阶 70  ██  ← 浅色主题主色常用
色阶 80  ██  ← 浅色容器常用
色阶 90  ██  ← 最亮背景色
色阶 95  ██  ← 表面背景
色阶 98  ██  ← 最亮
色阶 100 ██  纯白
```

### 1.3 色调用法速查

| 色阶 | 典型用途 |
|------|----------|
| 0 | 深色主题主色 |
| 10 | 深色主题主容器 |
| 20 | 深色主题次要元素 |
| 30 | 深色主题轮廓 |
| **40** | **浅色主题主色（Primary 的标准位置）** |
| 50 | 浅色主题次要主色 |
| 80 | 浅色主题主容器 |
| 90 | 最亮背景 |
| 95 | 页面背景（浅色主题） |
| 98 | 深色主题页面背景 |

## 二、主色、辅助色、错误色的生成规则

### 2.1 三角色阶体系

MD3 的每个角色使用同一色调的**两个色阶**：

- **ON 角色**（On-[Role]）：在角色颜色上方显示的文字/图标颜色，总是深色或浅色以确保对比度
- **角色颜色**（[Role]）：填充色/背景色
- **容器**（[Role] Container）：承载该角色内容的浅色容器

### 2.2 主色（Primary）

**生成规则：**
- 从用户指定的主色（陛下指定的 `#0071e3`）确定色调（Hue）和色相（Chroma）
- 主色通常使用该色调的 **40 号色阶**（浅色主题）或 **0 号色阶**（深色主题）
- ON-Primary 使用高对比度单色（白或接近白色）

```
浅色主题：
  Primary:         #0071e3  (色阶 40，Hue=Blue, Chroma=适中)
  On-Primary:      #ffffff  (白色，确保对比度)
  Primary-Container: #d3e4ff (色阶 80，更浅)
  On-Primary-Container: #001a40 (色阶 10，深蓝)

深色主题：
  Primary:         #a8c8ff  (色阶 80，发光感主色)
  On-Primary:      #003062  (色阶 10，深蓝)
  Primary-Container: #00468a (色阶 30)
  On-Primary-Container: #d3e4ff (色阶 90)
```

### 2.3 辅助色（Secondary）

**生成规则：**
- 通常与主色保持协调的色调关系（邻近色或互补色）
- 使用该色调的 **40 号色阶**
- 辅助容器比主容器更柔和

```
浅色主题：
  Secondary:         #565f71
  On-Secondary:      #ffffff
  Secondary-Container: #d9e3f8
  On-Secondary-Container: #131c2b

深色主题：
  Secondary:         #bec3d6
  On-Secondary:      #293042
  Secondary-Container: #404759
  On-Secondary-Container: #d9e3f8
```

### 2.4 错误色（Error）

**生成规则：**
- 固定使用 **Red 色调**，不能自定义色调
- 语义明确：传达错误、警告、危险状态
- 色阶 40 用于主错误色，色阶 80 用于容器

```
浅色主题：
  Error:              #ba1a1a
  On-Error:           #ffffff
  Error-Container:    #ffdad6
  On-Error-Container: #410002

深色主题：
  Error:              #ffb4ab
  On-Error:           #690005
  Error-Container:    #93000a
  On-Error-Container: #ffdad6
```

## 三、浅色容器（Container）和深色容器的对比

### 3.1 容器的作用

容器是承载内容的背景区域，比主色更柔和，用于：
- 卡片、按钮背景
- 输入框背景
- 列表项背景
- 弹出菜单背景

### 3.2 浅色容器 vs 深色容器

| 属性 | 浅色容器 (Light Container) | 深色容器 (Dark Container) |
|------|---------------------------|--------------------------|
| 色阶 | **80-90** | **10-30** |
| 颜色倾向 | 更亮、更淡 | 更暗、更饱和 |
| 与主色关系 | 主色的浅色调 | 主色的深色调 |
| 典型用途 | 浅色主题的卡片背景 | 深色主题的卡片背景 |
| 透明度 | 通常实色，无透明度 | 通常实色，无透明度 |

### 3.3 具体对比示例（以 Blue 为主色 #0071e3）

| 角色 | 浅色主题 | 深色主题 |
|------|----------|----------|
| Primary | `#0071e3` (40) | `#a8c8ff` (80) |
| Primary Container | `#d3e4ff` (80) | `#00468a` (30) |
| On Primary | `#ffffff` | `#003062` |
| On Primary Container | `#001a40` (10) | `#d3e4ff` (90) |

**关键规律：**
- 浅色主题：容器比主色更亮（色阶数更大）
- 深色主题：容器比主色更暗（色阶数更小）
- ON 颜色与容器形成强对比

## 四、亮色主题 vs 暗色主题的差异

### 4.1 整体结构对比

| 维度 | 浅色主题 (Light) | 深色主题 (Dark) |
|------|-----------------|-----------------|
| 表面背景色 | `#fffbfe` (色阶 98) | `#1c1b1f` (色阶 95 附近) |
| 主色明度 | 较暗 (色阶 40) | 较亮 (色阶 80+) |
| 辅助色明度 | 较暗 | 较亮 |
| 对比度要求 | WCAG AA+ (≥4.5:1) | WCAG AA+ (≥4.5:1) |
| 阴影 | 有阴影（Elevation） | 无阴影，用亮度区分层级 |

### 4.2 MD3 表面层级系统

MD3 用**亮度（Brightness）** 而非阴影来区分层级：

| 层级 | 浅色主题 | 深色主题 | 用途 |
|------|----------|----------|------|
| Surface | `#fffbfe` | `#1c1b1f` | 页面背景 |
| Surface Container Low | `#f7f2fa` | `#211f26` | 最低容器层 |
| Surface Container | `#ece6f0` | `#2b2930` | 标准容器层 |
| Surface Container High | `#e6e0ea` | `#36343b` | 较高容器层 |
| Surface Container Highest | `#e0dbe4` | `#3f3e45` | 最高容器层 |
| Surface Bright | `#fffbff` | `#49454f` | 高亮表面 |

### 4.3 暗色主题的特殊设计原则

1. **发光主色（Glow effect）**：深色主题的主色使用高色阶（80+），在暗背景上产生"发光"效果
2. **降低饱和度**：深色主题的辅助色通常降低饱和度以减少视觉疲劳
3. **轮廓使用**：在深色主题中，轮廓（Outline）色用于分隔线而非阴影
4. **无纯黑背景**：MD3 禁止使用 `#000000`，最低使用色阶 90-95 左右的深色

## 五、陛下指定的主色 #0071e3 在 MD3 体系中的位置

### 🔴P1 重点：#0071e3 = Blue 40（MD3 标准主色位置）

陛下指定的 `#0071e3` 正好位于 Material Design 3 体系的 **Blue 色调的第 40 号色阶**。

### 5.1 #0071e3 的身份确认

| 属性 | 值 |
|------|-----|
| Hex | `#0071e3` |
| 色调 (Hue) | Blue (约 230-240°) |
| 色相 (Chroma) | 适中 |
| MD3 色阶 | **40** ← **主色的标准位置** |
| 角色 | Primary（主色） |
| 对应深色主题主色 | `#a8c8ff` (Blue 80) |

### 5.2 从 #0071e3 生成的完整 Blue 调色板

```
Blue 0:    #000000    （MD3 深色主题主色）
Blue 10:   #001a40
Blue 20:   #002e6d
Blue 30:   #00468a    （深色主题主容器）
Blue 40:   #0071e3    ← 陛下指定的主色
Blue 50:   #2584ff
Blue 60:   #4d97ff
Blue 70:   #74abff
Blue 80:   #a8c8ff    （深色主题主色）
Blue 90:   #d3e4ff    （浅色主题主容器）
Blue 95:   #ecf4ff
Blue 98:   #f6faff
Blue 100:  #ffffff
```

### 5.3 在 MD3 色彩工具中的位置

在 [Material Design Color Tool](https://material.io/resources/color) 中：
- 选择 **Primary** 角色
- 拖动色调轮到 **Blue** 区域
- 系统自动选中 **40** 号色阶
- 输入 Hex 值 `#0071e3` 可精确匹配 MD3 Blue 40

## 七、MD3 色彩系统的数学生成原理

### 7.1 从主色生成调色板的算法

MD3 调色板通过以下步骤从单一主色生成：

1. **确定色调 (Hue)**：用户在色轮上选择，或从 Hex 值提取
2. **计算基础色相 (Chroma)**：根据主色的饱和度计算
3. **应用色调映射表**：每种色调有预定义的最大色相值
4. **生成 13 级明度**：从最暗(0)到最亮(100)，使用线性或曲线插值
5. **计算 ON 颜色**：自动生成高对比度的文字颜色

### 7.2 色调映射表（部分）

| 色调 | 基准 Hue | 最大 Chroma |
|------|----------|-------------|
| Red | 0° | 87 |
| Orange | 30° | 97 |
| Yellow | 60° | 94 |
| Lime | 90° | 90 |
| Green | 120° | 87 |
| Cyan | 180° | 88 |
| **Blue** | **240°** | **87** |
| Purple | 270° | 85 |
| Magenta | 300° | 84 |
