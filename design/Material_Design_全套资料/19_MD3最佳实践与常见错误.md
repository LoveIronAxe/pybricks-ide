# Material Design 3 最佳实践与常见错误

## 一、最佳实践

### 1. 色彩使用
- 主色（Primary）不超过页面面积的 40%，保持视觉平衡
- 强调色（Accent/Tertiary）仅用于需要突出交互的关键元素
- 使用 MD3 调色板系统生成和谐的色彩搭配，而非手动选择随机颜色
- 语义化用色：成功用绿色、警告用橙色、错误用红色

### 2. 圆角统一
- 同类组件必须使用相同的圆角值，保证视觉一致性
- 常用圆角规范：
  - 小组件（Checkbox、Chip、小按钮）：4px / 8px
  - 中组件（卡片、输入框）：8px / 12px
  - 大组件（弹窗、Sheet）：12px / 16px / 28px（MD3 大圆角特色）
- 同一种组件永远使用同一个圆角值，不要在同一组件的不同实例间混用

### 3. 间距一致性
- 严格使用 8dp 网格系统，所有间距必须是 8 的倍数
- 常用间距值：4dp / 8dp / 16dp / 24dp / 32dp / 48dp
- 禁止随意设定 10dp、13dp、15dp 等非网格间距值
- 组件内部间距和组件间间距分开考虑

### 4. 阴影使用
- MD3 倾向**不使用阴影**，而是用背景色区分层次
- elevation 在 MD3 中更多通过色差（Surface Tint）而非阴影实现
- 如果必须使用阴影，值要极轻（MD3 原生 elevation 通常 0-3dp）
- 避免多层阴影、避免大范围阴影覆盖

### 5. 字体层级
- 保持 4-5 级字体层级就够了，不要过度细分
- 建议层级结构：
  - 标题（Headline）：28-32sp
  - 副标题（Title）：20-24sp
  - 正文（Body）：14-16sp
  - 辅助文字（Caption）：12sp
  - 按钮文字（Label）：14sp（首字母大写，非全大写）
- 同一种文本类型只对应一种样式，不要混用

### 6. 暗色主题
- **不是简单反色**！这是最常见的暗色主题误区
- 正确的暗色主题做法：
  - 降低饱和度（避免刺眼的高亮色）
  - 提高对比度（确保文字可读性）
  - 背景色使用深灰而非纯黑（如 #1C1B1F 而不是 #000000）
  - 表面色使用递进的灰阶（如 Surface 0 / Surface 1 / Surface 2 / Surface 3）
  - 主色适当提亮以匹配暗色背景
- 使用 MD3 的暗色主题色彩系统，不要手动推导

## 二、常见错误

### ❌ 圆角过大或过小
| 组件 | 错误做法 | 正确做法 |
|------|---------|---------|
| 小按钮 | 4px（太小，像旧版） | 20px（全圆角，MD3 风格）或 8px |
| 弹窗/Dialog | 12px（MD2 遗留） | 28px（MD3 大圆角特色） |
| 卡片 | 混用 4px 和 16px | 统一 12px |
| 输入框 | 4px | 4px / 8px |

### ❌ 颜色对比度不够
- 文字与背景的对比度必须达到 WCAG AA 标准（4.5:1 正常文字，3:1 大文字）
- MD3 调色板中的 `on-surface-variant` 专门为此设计，不要随意替换
- 深色背景上的浅色文字尤其要注意对比度

### ❌ 滥用阴影
- MD3 风格偏好 **无阴影**，用背景色区分层次
- 常见错误：在卡片上叠加多层阴影，试图营造立体感
- 正确做法：用 Surface Tint（表面着色）模拟 elevation，或使用边框线

### ❌ 字体大小不统一
- 同一个 UI 中出现 12sp / 13sp / 14sp / 15sp 混用
- 导致字体大小不统一的原因：
  - 从设计稿复制数值时未做规范检查
  - 不同设计师使用不同规范
- 建议：建立字体规范文档，严格执行

### ❌ 按钮文字用纯大写
- **MD3 按钮文字是首字母大写（Sentence case），不是 ALL CAPS（全大写）**
- 错误：`SUBMIT`、`CANCEL`
- 正确：`Submit`、`Cancel`
- MD3 认为 ALL CAPS 会降低阅读流畅性，且不符合自然语言习惯

## 三、对本项目 IDE 的具体建议

### 侧边栏：使用 Navigation Rail 规范
- 宽度固定为 **56dp**，图标+标签模式可扩展到 80dp
- 图标大小：24dp
- 图标与文字间距：12dp
- 选中态使用 Surface Tint 背景色（非阴影）
- 侧边栏背景使用 `Surface` 颜色，与主内容区形成层次

### 列表项 hover/active 效果
- **用背景色变化，不用阴影**
- 推荐做法：`Surface Variant` 色做 hover 背景，`Secondary Container` 色做选中背景
- 避免：hover 时添加 box-shadow

### 弹窗遮罩（Dialog Overlay）
- **用半透明白色背景 + blur，不要用深色遮罩**
- 推荐：白色 50% 透明度 + backdrop-filter: blur(8px)
- 错误：黑色 70% 透明度（过于压抑）
- Dialog 本体使用白色/深色 Surface，圆角 28px

### 其他 IDE 特定建议
- 窗口最小尺寸建议：360dp × 540dp（移动端）/ 600dp × 480dp（桌面端）
- 编辑器区域使用暗色主题，工具栏使用亮色主题时注意边界分明
- 代码编辑器的行高建议 1.5 倍字体大小
- 标签页（Tab）使用 MD3 Tabs 规范：文字首字母大写，下划线指示器

## 四、🔴 P1 重点：本项目 IDE 具体应用的 10 条最佳实践清单

> 以下清单为本项目 PyBricks IDE 实现 Material Design 3 风格的具体规范，优先级 P1，必须逐一落实。

1. **侧边栏宽度锁定为 56dp**（Navigation Rail），图标 24dp，选中态用背景色不用阴影

2. **所有间距必须是 8dp 倍数**：组件内 4/8dp，组件间 16/24/32dp，禁止出现 10dp、13dp、15dp 等数值

3. **圆角统一规范**：
   - 按钮：20px（全圆角 MD3 风格）
   - 卡片/列表项：12px
   - 弹窗/Sheet：28px
   - 同一组件的圆角值全局一致，禁止混用

4. **主色占比不超过 40%**：主色仅用于侧边栏选中态、主要按钮、关键图标，其他区域使用 `Surface` 和 `Surface Variant`

5. **hover/active 效果用背景色变化实现**：列表项 hover 用 `Surface Variant`，选中用 `Secondary Container`，完全不用 box-shadow

6. **弹窗遮罩用白色半透明 + blur（8px）**：禁止使用深色遮罩，Dialog 圆角 28px，标题使用 Title Large 字体（22sp）

7. **字体层级控制在 4-5 级**：标题 22sp、正文 14sp、辅助 12sp、按钮 14sp 首字母大写，禁止在同一组件内出现 3 种以上字体大小

8. **暗色主题正确实现**：
   - 背景用深灰（#1C1B1F）而非纯黑
   - 主色提亮以适配暗色背景
   - 保持足够对比度（WCAG AA）
   - 不是简单反色，而是降低饱和度+提高对比度

9. **按钮文字使用首字母大写（Sentence case）**：`Submit` `Cancel` `Save`，禁止全大写 `SUBMIT` `CANCEL`

10. **无障碍对比度合规**：所有文字与背景的对比度 ≥ 4.5:1，使用 MD3 调色板中的语义化颜色（`on-surface` / `on-surface-variant`）而非手动指定十六进制颜色值
