# 01_官方原则与设计理念.md

> 一句话概括：Material Design 3 (MD3) 是 Google 的最新设计系统，以 Material You（动态颜色）、自适应设计和情感化体验为核心，支持跨平台构建高质量产品。

---

## 🔴P1 本项目重点

- **动态颜色（Dynamic Color）**：IDE 可从用户壁纸或内容中提取主色调，自动生成协调的亮/暗主题配色，**极大提升个性化体验**
- **自适应布局**：MD3 组件天然支持响应式设计，IDE 各面板（代码区、侧边栏、终端）可利用栅格系统和间距 token 保持一致性
- **设计 Token 体系**：颜色、字体、间距、圆角、阴影均有 token 化定义，IDE 主题可基于 token 快速切换
- **M3 Expressive 更新（2025.5）**：按钮等组件新增更多尺寸（extra small ~ extra large）、方形/圆形可变形状、toggle 变体；IDE 可利用这些新特性提升专业感

## 核心理念

### 1. Material You（动态颜色）
Material You 是 MD3 的标志性特性，允许从用户 wallpaper 或应用内容中提取单个颜色，自动生成完整的可访问配色方案，并应用到所有 UI 元素上。

**核心机制：**
- 从壁纸/内容提取"源色"（source color）
- 通过色相轮映射生成 Primary、Secondary、Tertiary、Error、Surface 等角色
- 同时生成 Light 和 Dark 主题方案
- 所有色彩映射满足 WCAG 可访问性对比度要求

**IDE 应用建议：**
- 提供「跟随系统主题」选项
- 提供「自定义主色」选项，用户可选择强调色

### 2. 自适应设计（Adaptive Design）
MD3 优先考虑在手机、平板、桌面等不同屏幕尺寸上的体验一致性。

**关键原则：**
- **栅格系统**：以 4dp/8dp 为基础单位，IDE 侧边栏宽度、代码区边距等应为此的倍数
- **组件尺寸响应**：同一组件在不同屏幕上有不同尺寸推荐（如 Navigation drawer vs Navigation rail）
- **窗口大小断点**：
  - 0-599dp：手机竖屏
  - 600-839dp：手机横屏/小平板
  - 840-1199dp：平板/小桌面
  - 1200dp+：桌面

### 3. Expressive（情感化）更新
M3 Expressive 是 2025 年 5 月的重大更新，将 MD3 从「功能导向」升级为「情感化设计」：

- **新组件**：新增 15 个或更新组件，包括更丰富的尺寸、形状、功能
- **强调字体**：扩展的 M3 字体系统，新增强调样式，支持使用 variable font 打造编辑感
- **Motion 物理系统**：简化的 spring-based 动效系统，交互更自然、流畅
- **扩展形状库**：35 种形状，可用于装饰细节和形状变换

## 设计原则（Design Principles）

### 1.  Material（材质隐喻）
MD3 延续了"材质"隐喻，用**高度（elevation）**和**表面（surface）**的视觉变化来表达层级关系：
- 阴影和高度用于区分内容层级
- 表面颜色通过 tint（微调）和 shade（阴影）区分层级

### 2. 颜色系统（Color System）

**角色色（Role Colors）：**
| 角色 | 用途 | 示例值（默认主题） |
|------|------|------------------|
| Primary | 主品牌色、主要操作 | `#6442d6` |
| On Primary | Primary 上的文字/图标 | `#ffffff` |
| Primary Container | Primary 的浅色容器 | `#9f86ff` |
| Secondary | 次要操作、次要品牌 | `#5d5d74` |
| Surface | 卡片/面板背景 | `#fefbff` |
| Surface Variant | 分组/区分背景 | `#e8e0e8` |
| On Surface | Surface 上的文字 | `#1c1b1d` |
| Outline | 边框、分割线 | `#787579` |
| Error | 错误状态 | `#ff6240` |

**IDE 配色建议：**
- 代码区背景：使用 Surface 或 Surface Variant
- 侧边栏背景：使用 Surface 或 Surface Container（MD3 新增）
- 强调色：使用 Primary Container 而非纯 Primary（更柔和）
- 暗色主题：所有颜色角色自动反转，Surface 变深

### 3. 字体系统（Typography）

MD3 使用 **Google Sans** 系列字体，包括：
- **Display**：大标题（96/88/57/45/36sp）
- **Headline**：中等标题（32/28/24sp）
- **Title**：卡片/列表标题（22/16/14sp，Title Medium 权重 500）
- **Body**：正文（16/14/12sp）
- **Label**：按钮/标签（14/12/11sp，Label Medium 权重 500）

**字体变量（Font Variation）：**
- `GRAD`（字重梯度）：控制字重
- `opsz`（字号）：根据实际显示大小优化视觉权重

**IDE 字体建议：**
- 代码字体：使用 Google Sans Mono（MD3 专用等宽字体）
- 界面字体：Google Sans Text
- 代码区字号建议：14-16sp（Code M）
- 标签/按钮：Label M（12sp，500）

### 4. 高度与阴影（Elevation）

MD3 组件有 3 级 elevation：
```
Elevation 1: 0px 1px 2px rgb(0 0 0 / 30%), 0px 1px 3px 1px rgb(0 0 0 / 15%)
Elevation 2: 0px 1px 2px rgb(0 0 0 / 30%), 0px 2px 6px 2px rgb(0 0 0 / 15%)
Elevation 3: 0px 1px 3px rgb(0 0 0 / 30%), 0px 4px 8px 3px rgb(0 0 0 / 15%)
```

### 5. 圆角（Shape）

- **小组件**（chips、buttons）：完全圆角（full rounded）
- **中等组件**（cards、dialogs）：一般圆角（medium）
- **大方形组件**（sheets）：较小圆角（small）
- 默认圆角单位：4dp（small）、8dp（medium）、12dp（large）

### 6. 动效原则

MD3 强调**有意义的动效**：
- **Entrance**：元素进入时的渐现 + 轻微上移
- **Exit**：元素离开时的渐隐
- **State changes**：状态变化（如按钮点击）使用 spring 物理动画
- **Duration**：
  - 短：100-200ms（微小交互）
  - 中：300ms（标准过渡）
  - 长：500ms（页面级变化）
- **Easing**：推荐使用 spring 或标准 Material 缓动曲线
