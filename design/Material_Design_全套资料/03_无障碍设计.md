# 03_无障碍设计.md

> 一句话概括：MD3 内置可访问性支持，包括 WCAG AA 标准对比度、触摸目标尺寸、焦点管理和动态字体支持，确保所有用户可用。

---

## 🔴P1 本项目重点（IDE 无障碍设计）

### 1. 颜色对比度（WCAG AA/AAA）

**文字对比度要求：**
| 级别 | 文字大小 | 最小对比度 |
|------|---------|-----------|
| WCAG AAA | 18sp+ 或 14sp bold | **7:1** |
| WCAG AA | 正常文字（< 18sp） | **4.5:1** |
| WCAG AA | 大文字（18sp+ 或 14sp bold） | **3:1** |
| MD3 建议 | 界面元素（图标、边框） | **3:1** |

**IDE 实践：**
- 代码编辑区文字（白底黑字）：21:1 ✅ 远超标准
- 暗色主题代码文字（黑底白字）：21:1 ✅
- 按钮文字 vs 按钮背景：需确保 ≥ 4.5:1
- 工具栏图标 vs 背景：≥ 3:1
- **动态颜色场景**：MD3 自动保证所有映射色满足对比度，IDE 自定义主题时需人工验证

### 2. 触摸/点击目标尺寸

**MD3 规范：**
- 最小触摸目标：**44 × 44 dp**（约 48px）
- 推荐图标按钮尺寸：**40 × 40 dp** 起
- 按钮内文字周围 padding：**8dp** 起

**IDE 实践：**
- 工具栏图标按钮：最小 **40dp × 40dp**
- 列表项行高：至少 **48dp**
- 右键菜单项高度：至少 **44dp**
- 代码编辑区点击目标（折叠区、gutter）：确保足够大

### 3. 焦点状态（Focus State）

MD3 为所有可交互元素定义了清晰的焦点状态：

**焦点环规范：**
- 颜色：使用 `on-surface` 色，带 **2dp** 偏移
- 焦点环宽度：**3dp**（IDE 建议至少 2dp）
- 焦点环样式：**方形/圆形**均可，与组件形状匹配
- 样式：虚线框或实线框均可，**视觉上突出但不刺眼**

**焦点遍历顺序：**
- 符合逻辑阅读顺序（DOM 顺序）
- Tab 键遍历顺序：工具栏 → 编辑区 → 侧边栏
- 所有交互元素均需可聚焦（使用 `tabindex`）

**IDE 实践：**
- 代码编辑区可聚焦（`contenteditable`）
- 工具栏按钮使用 `<button>` 元素，天然可聚焦
- 侧边栏项添加 `role="button"` + `tabindex`
- 不要用 `outline: none` 取消焦点环（无障碍灾难）

### 4. 字体大小与可缩放性

**MD3 字体系统：**
- 支持用户系统级字体缩放（Android / iOS / macOS）
- 字体大小使用 `sp`（可缩放像素）而非 `dp`
- **最小可读字体**：Label 级别 11sp，但 WCAG AA 要求实际渲染 ≥ 12sp

**IDE 实践：**
- 代码字体：14-16sp（`--code-font-size`）
- 界面文字：12-14sp（`--ui-font-size`）
- 允许用户调整代码区字号（≥ 10sp）
- **支持 Ctrl/Cmd + 滚轮缩放**
- **尊重系统字体大小设置**（macOS Accessibility / Windows 显示设置）

### 5. 动态字体（Variable Fonts）与可访问性

MD3 使用 Google Sans 系列字体，支持 `font-variation-settings`：
- `GRAD`：控制字重梯度
- `opsz`：字号相关的光学修正（大字号自动调整字重以保持可读性）

IDE 自定义字体时，优先选择支持 variable font 的等宽字体。

### 6. 语义化标签与 ARIA

**MD3 组件默认语义化，IDE 实现时需保持：**
- 按钮用 `<button>` 而非 `<div>` + click handler
- 导航用 `<nav>` + `role="navigation"`
- 列表用 `<ul>/<li>` 而非 `<div>` 堆叠
- 开关用 `<input type="checkbox/switch">` 或 ARIA `role="switch"`

**ARIA 关键属性：**
```html
<!-- 切换按钮 -->
<button aria-pressed="true/false" aria-label="加粗">

<!-- 标签页 -->
<div role="tablist">
  <button role="tab" aria-selected="true" aria-controls="panel1">文件</button>
  <button role="tab" aria-selected="false" aria-controls="panel2">编辑</button>
</div>

<!-- 对话框 -->
<div role="dialog" aria-modal="true" aria-labelledby="dialog-title">
```

### 7. Motion 与前庭障碍支持

MD3 的 Motion 物理系统：
- Spring-based 动画比传统 ease 更自然
- **尊重「减弱动态效果」系统设置**（iOS Reduce Motion / macOS Accessibility）

**IDE 实践：**
- 所有动画使用 `prefers-reduced-motion` 媒体查询检测：
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
- 或者在 JS 中检测：`window.matchMedia('(prefers-reduced-motion: reduce)')`

### 8. 色彩无障碍（Color Blindness）

MD3 的动态颜色系统**自动考虑色彩障碍用户**：
- 颜色映射时使用色调（hue）+ 明度（lightness）双重区分
- 不依赖单一颜色传达信息（配合图标、文字、图案）

**IDE 实践：**
- 错误/警告状态：不要只用红色（`#ff6240`），加 ❌ / ⚠️ 图标或文字
- Git 分支颜色：除颜色外显示分支名
- 文件状态（新增/修改/删除）：颜色 + 图标 + 文字三重标识

### 9. 组件无障碍规格速查

| 组件 | 触摸目标 | 对比度 | 焦点要求 |
|------|---------|--------|---------|
| Button | ≥ 40dp 高，min-width 48dp | 文字 ≥ 4.5:1 | 可聚焦，Tab 遍历 |
| Icon button | ≥ 40dp × 40dp | 图标 ≥ 3:1 | `aria-label` 说明 |
| Text field | 高度 ≥ 56dp | 边框/文字 ≥ 4.5:1 | 焦点 + `aria-label` |
| Switch | 高度 ≥ 40dp | 轨道/把手 ≥ 3:1 | `role="switch"` |
| Checkbox | ≥ 40dp × 40dp | 勾选符号 ≥ 3:1 | `role="checkbox"` |
| Slider | 轨道高度 ≥ 4dp，thumb ≥ 20dp | 值显示 ≥ 4.5:1 | 键盘方向键可调 |
| Menu item | 高度 ≥ 48dp | 文字 ≥ 4.5:1 | 键盘上下选择 |
| Tabs | 高度 ≥ 48dp | 选中标签 ≥ 4.5:1 | `role="tab"` + 方向键 |

### 10. 键盘导航支持

**最低要求（IDE 无障碍基础）：**
| 快捷键 | 功能 |
|--------|------|
| `Tab` | 切换焦点到下一个元素 |
| `Shift+Tab` | 切换焦点到上一个元素 |
| `Enter` / `Space` | 激活按钮/选择项 |
| `方向键` | 在组件内导航（tabs、menus、lists） |
| `Escape` | 关闭对话框/菜单/面板 |
| `Alt+F4` / `Cmd+W` | 关闭窗口 |

**增强要求：**
- `F6`：在面板间循环切换焦点
- `Ctrl+Shift+Tab`：切换焦点到上一个面板
- 高亮当前焦点元素（焦点环）
